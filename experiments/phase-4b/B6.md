# B6: Hooks（lint 自動実行）の品質影響測定

## 仮説

PostToolUse Hooks で ESLint + Prettier を自動実行すると、Hooks なしと比べて lint エラーが **50% 以上** 減少する。

- 元調査: [04-slash-commands.md](/experiments/claude-code-prompting/04-slash-commands) — 「Hooks は決定論的（deterministic）、CLAUDE.md は助言的（advisory）」

## 実験条件

| 項目 | 値 |
|------|-----|
| 独立変数 | PostToolUse Hooks の有無（2 条件） |
| モデル | claude-sonnet-4-20250514 |
| タスク | `validateEmail` 関数の追加 + テスト（B1 と同一） |
| CLAUDE.md | 共通（事実形式、30 行、lint/format コマンド記載） |
| 試行数 | 各条件 7 回（合計 14 回） |

### 2 条件の違い

| 条件 | 設定 |
|------|------|
| no-hooks | `.claude/settings.json` に hooks 設定なし |
| hooks | Edit/Write 後に `eslint --fix` + `prettier --write` を自動実行 |

### Hooks 設定（hooks 条件）

```json
{
  "hooks": {
    "PostToolUse": [{
      "matcher": "Write|Edit",
      "hooks": [{
        "type": "command",
        "command": "npx eslint --fix \"$file\" && npx prettier --write \"$file\""
      }]
    }]
  }
}
```

## 判定: 仮説支持

Hooks 条件は全 7 試行で ESLint エラー **0 件**（完全排除）。no-hooks 条件は 7 試行中 3 試行でエラーが発生（発生率 43%、平均 1.0 件）。Hooks は lint エラーを **100% 削減**し、基準の 50% を大幅に超えた。

## 考察

- **決定論的メカニズムの威力**: Hooks は Claude の判断に依存せず、Write/Edit 後に機械的に `eslint --fix` を実行する。助言的な CLAUDE.md の記述とは本質的に異なるアプローチで、100% の品質保証を実現
- **影響範囲は lint/format に限定**: テスト通過率・型エラーは両条件とも同一。Hooks の効果は lint/format という決定論的に検証可能な領域に限られ、テスト設計やロジックの正確性には影響しない
- **Prettier 不整合は両条件ともゼロ**: Claude は Prettier のフォーマットルールには元々従う傾向が強い。Hooks の主効果は ESLint のより細かいルール（未使用変数、import 順序など）への対応
- **Phase 4B 全体で唯一仮説を支持した実験**: B1〜B5 が不支持だったなかで、Hooks だけが明確な効果を示した。「CLAUDE.md の書き方を工夫するよりも Hooks で自動化するほうが確実」という実践的な示唆を提供

### 推奨設定

```json
{
  "hooks": {
    "PostToolUse": [{
      "matcher": "Write|Edit",
      "hooks": [{
        "type": "command",
        "command": "npx eslint --fix \"$CLAUDE_FILE_PATH\" && npx prettier --write \"$CLAUDE_FILE_PATH\""
      }]
    }]
  }
}
```
