# B3: CLAUDE.md 行数と遵守率の関係

## 仮説

CLAUDE.md の行数が増えるほど、コアルールの遵守率が低下する。30 行と 200 行超では有意な差が生じる。

- 元調査: [02-claude-md.md](/experiments/claude-code-prompting/02-claude-md)（200 行上限）、[14-anti-patterns.md](/experiments/claude-code-prompting/14-anti-patterns)（詰め込みアンチパターン）、[15-claude-md-examples.md](/experiments/claude-code-prompting/15-claude-md-examples)（30-100 行が理想）

### 3 条件の違い

| 条件 | 行数 | 内容 |
|------|------|------|
| short | 23 行 | コアルール 10 個 + 開発コマンドのみ |
| medium | 96 行 | short + アーキテクチャ説明 + 注意事項 + 具体例 |
| long | 247 行 | medium + ファイル個別説明 + API ドキュメント + 冗長な補足 |

3 条件すべてにコアルール 10 個（B1 と同一）を含む。追加情報がコアルールの遵守を邪魔するかを検証。

## 判定: 仮説不支持（逆方向の結果）

仮説は「行数増加 → 遵守率低下」だったが、実際は逆で **long（247 行）が最高スコア**（9.86）を記録。7 試行中 6 試行が 10/10 の完全スコア。

## 考察

- **長い CLAUDE.md は「ノイズ」ではなく「追加コンテキスト」**: 具体例・アンチパターン・ファイル別解説を含む長い CLAUDE.md が最も高い遵守率を示した
- **r6（関数サイズ）が最も顕著な差**: short 57% → medium 29% → long 86%。medium で低下した後 long で大幅回復。long 条件にはファイルサイズに関する具体的な例示が含まれており、これが遵守を促進した可能性が高い
- **「短く書くべき」という助言は過度に単純**: コミュニティで広く語られる「CLAUDE.md は 30-100 行に収めるべき」という助言は、少なくとも 250 行程度までは当てはまらない。重要なのは行数ではなく、ルールの具体性と例示の質
