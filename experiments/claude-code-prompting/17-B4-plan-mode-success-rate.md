# B4: Plan mode の初回成功率測定

## 仮説

Plan mode を使用すると、中規模タスク（3-5 ファイル変更）の初回成功率が有意に向上する。
ある報告では 37% 向上とされるが、再現性は不明。

- 元調査: [03-plan-mode.md](03-plan-mode.md)、[10-task-new-feature.md](10-task-new-feature.md)
- 検証対象: [A3 段階的ワークフローガイド](../../guides/workflow.md) の Plan mode 推奨

---

## 実験デザイン

### 独立変数（操作する変数）

2 要因:

| 要因 | 条件 |
|------|------|
| Plan mode | A: なし（直接実装）/ B: あり（計画→実装の 2 段階） |
| モデル | Sonnet / Opus |

合計 4 条件: Sonnet×Plan なし, Sonnet×Plan あり, Opus×Plan なし, Opus×Plan あり

### 従属変数

| 指標 | 定義 | 測定方法 |
|------|------|---------|
| 初回成功率 | テスト + 型チェック + lint がすべて通るか | `vitest run && tsc --noEmit && eslint src/ tests/` |
| 所要ターン数 | Claude のツール呼び出し回数 | JSON 出力からカウント |
| 修正回数 | エラー発生後の修正サイクル数 | JSON 出力から修正ループを検出 |

### 統制変数

| 項目 | 値 |
|------|-----|
| タスクプロンプト | 固定文言（下記） |
| CLAUDE.md | 共通（事実形式、30 行） |
| 反復回数 | 各条件 5 回（合計 20 回） |

```

1. order.ts の createOrder 関数で、注文作成後に通知を送信する
2. user.ts の getUserById で注文者の email を取得する
3. notification.ts の sendNotification を使って通知を送る
```

## 実験手順

### Plan なし（条件 A）

1. `sample-project-clean/` から復元
2. 共通 CLAUDE.md を配置
3. `claude -p "プロンプト" --model {model} --output-format json` を実行
4. 判定スクリプトで成功/失敗を判定
5. JSON 出力からターン数・修正回数を抽出

### Plan あり（条件 B）

1. `sample-project-clean/` から復元
2. 共通 CLAUDE.md を配置
3. Step 1: `claude -p "計画プロンプト" --model {model} --output-format text` で計画を取得
4. Step 2: 計画テキストを含む実装プロンプトで実行
5. 判定・データ抽出は条件 A と同じ

---

## 判定基準

**初回成功の定義**: Claude の最初の実装完了時点（「完了しました」等の宣言後）で以下がすべて通ること:

```bash
npx vitest run          # テスト全通過
npx tsc --noEmit        # 型エラーなし
npx eslint src/ tests/  # lint エラーなし
```

**機能要件チェック**（手動 or スクリプト）:
1. `createOrder` が `sendNotification` を呼んでいる
2. 通知の宛先がユーザーの email である
3. 件名に orderId が含まれる
4. テストが通知送信を検証している

---

## 予想される結果

- 仮説支持: Plan ありの初回成功率が Plan なしを 20% 以上上回る
- 所要ターン数は Plan ありの方が少ない（計画で手戻りを減らせる）
- Opus は Sonnet より全体的に成功率が高い（モデル性能の差）

## コスト見積もり

| 項目 | Sonnet | Opus |
|------|--------|------|
| 1 回あたり推定 | $0.15-0.30 | $0.75-1.50 |
| 10 回（Plan あり/なし各 5 回） | $1.5-3 | $7.5-15 |
| **合計** | **$9-18** | |

B4 は全実験中最もコストが高い。Opus の試行回数は結果を見ながら調整する。
