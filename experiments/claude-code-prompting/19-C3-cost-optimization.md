## 1. モデルの使い分け

### モデル特性と推奨用途

| モデル | 入力 | 出力 | 推奨用途 | 使用比率目安 |
|--------|------|------|---------|------------|
| **Opus 4.6** | $5/MTok | $25/MTok | 複雑なアーキテクチャ設計、マルチステップ推論、難易度の高いデバッグ | 5-10% |
| **Sonnet 4.6** | $3/MTok | $15/MTok | 日常のコーディング作業全般。コード生成、データ分析、ツール利用 | **80%以上（デフォルト推奨）** |
| **Haiku 4.5** | $1/MTok | $5/MTok | 単純な分類、要約、サブエージェントタスク、リアルタイム処理 | 10-15% |

### opusplan 戦略

`claude --model opusplan` でプランモード時は Opus、実行モード時は Sonnet に自動切り替え。計画に深い推論を、実装にコスト効率を両立するハイブリッドアプローチ。

### Effort レベル

`/effort low|medium|high|max` で推論の深さを調整可能。

| レベル | 特徴 |
|--------|------|
| low | 高速・低コスト。単純な質問向け |
| medium | デフォルト（Pro/Max ユーザー） |
| high/max | 深い推論。複雑な問題向けだがトークン消費大 |

## 2. Fast mode

Fast mode は Opus 4.6 の推論を 2.5 倍高速化する機能（研究プレビュー段階）。

| 項目 | 詳細 |
|------|------|
| 仕組み | 同じ Opus 4.6 モデル。バックエンドの推論設定を速度優先に変更。品質・能力は同一 |
| 料金 | **標準 Opus の 6 倍**（入力 $30/MTok、出力 $150/MTok） |
| 有効化 | `/fast` コマンドでトグル。`↯` アイコンで状態表示 |
| 課金 | サブスクリプションプランでも **extra usage（追加課金）扱い** |
| 使いどころ | ライブデバッグ、高速イテレーション、締め切りの厳しい作業 |
| 避けるべき場面 | 長時間の自律タスク、バッチ処理、コスト重視の作業 |

**注意**: セッション途中で有効化すると、既存の会話コンテキスト全体が Fast mode 料金で再課金される。セッション開始時に有効化する方がコスト効率が良い。

## 3. トークン消費の最適化

以下の戦略で **40-70% のコスト削減が実現可能**（コミュニティ報告）。

### コンテキスト管理

| 手法 | 効果 |
|------|------|
| `/clear` | タスク切り替え時にコンテキストをリセット。古いコンテキストは後続メッセージ全てでトークンを浪費 |
| `/compact` | マイルストーン完了後に定期的に実行。要約により不要情報を削除。`/compact Focus on code samples` のように保持内容を指示可能 |

### ファイル読み込みの回避

| 手法 | 効果 |
|------|------|
| `.claudeignore` | 不要ディレクトリを除外（`node_modules`, `dist`, `.next` 等）。Next.js で `.next/` 追加だけで **30-40% のコンテキスト削減** |
| 具体的なパス指定 | "Find where we handle auth" ではなく "Check `src/auth/middleware.ts`" |

### CLAUDE.md の最適化

- セッション開始時に必ず読み込まれるため、肥大化はセッション全体のコストに影響
- 200 行以下に保ち、専門的な指示は Skills に移動（オンデマンドロード）
- Skills への移動で **約 15,000 トークン/セッションの節約（82% 改善）**

### その他のテクニック

- Plan mode（Shift+Tab）で実装前に計画を立て、手戻りを防止
- サブエージェントに冗長な処理（テスト実行、ログ解析）を委譲
- Hooks でデータを前処理（例: 10,000 行ログから ERROR 行のみ抽出）
- MCP サーバーは使わないものを無効化。CLI ツール（`gh`, `aws`）の方がコンテキスト効率が良い
- 「説明不要、コードだけ出力して」で **出力トークンを 60-70% 削減**

## 4. 料金体系

### サブスクリプションプラン

| プラン | 月額 | Claude Code | 使用量目安 | デフォルトモデル |
|--------|------|-------------|-----------|--------------|
| Pro | $20 | 利用可 | ~44K トークン/5時間窓 | Sonnet 4.6 |
| Max 5x | $100 | 利用可 | ~88K トークン/5時間窓 | Opus 4.6 |
| Max 20x | $200 | 利用可 | ~220K トークン/5時間窓 | Opus 4.6 |
| Team Standard | $20/席 | 利用不可 | — | — |
| Team Premium | $100/席 | 利用可 | 5x Standard | Opus 4.6 |
| Enterprise | カスタム | 利用可 | カスタム | — |

- Pro の年間払いで約 15% 割引（実質 $17/月）
- Max プランは API 直接利用の **約 18 倍安い**（等価使用量で比較）

### API 従量課金

| モデル | 入力 | 出力 | キャッシュ読取 |
|--------|------|------|--------------|
| Opus 4.6 | $5/MTok | $25/MTok | $0.50/MTok |
| Sonnet 4.6 | $3/MTok | $15/MTok | $0.30/MTok |
| Haiku 4.5 | $1/MTok | $5/MTok | $0.10/MTok |

- **プロンプトキャッシュ**: ヒットは入力価格の 10%
- **Batch API**: 入出力ともに 50% 割引（24時間以内の処理）
- **1M コンテキスト**: 200K 超でも追加料金なし

### プラン選択の目安

| 月額 API コスト | 推奨プラン | 節約率 |
|----------------|-----------|--------|
| $50 未満 | Pro ($20) or 従量課金 | — |
| $100-300 | Max 5x ($100) | 中 |
| $500 以上 | Max 20x ($200) | **60-94%** |

実績: 8ヶ月で 10B トークン使用したユーザーが API 換算 $15,000 → Max プランで $800（**93% 節約**）。

## 5. コストモニタリング

### 組み込みコマンド

| コマンド | 対象 | 内容 |
|---------|------|------|
| `/cost` | API ユーザー | 現在セッションのトークン使用量と推定コスト |
| `/stats` | サブスクリプション | 使用パターンの表示 |
| `/context` | 全ユーザー | コンテキスト窓の使用状況（カテゴリ別内訳） |

### サードパーティツール

| ツール | 特徴 |
|--------|------|
| **ccusage** | CLI ベースの使用量分析。`~/.claude/` の JSONL ファイルを解析。日次/週次/月次レポート、5時間ブロック追跡 |
| **Bifrost** | LLM Gateway。Prometheus メトリクス、仮想キーによるチーム別コスト帰属、階層的予算制御 |
| **LiteLLM** | Python ベースプロキシ。100以上のプロバイダ対応、キー別支出追跡 |
| **Anthropic Console** | ワークスペース支出制限、Usage and Cost API |

### チーム向け予算管理

- Workspace spend limits でチーム全体の上限設定
- Workspace rate limit で Claude Code の API 使用量を制限（他ワークロード保護）
- Bedrock/Vertex 環境では LiteLLM 等のゲートウェイで支出追跡

## 6. コミュニティの知見

### 実際のコスト事例

| 指標 | 値 |
|------|------|
| 平均コスト | $6/開発者/日（90% が $12/日未満）— Anthropic 公式データ |
| 月額目安 | Sonnet 4.6 使用で $100-200/開発者/月 |
| パワーユーザー | 大規模リファクタやマルチエージェントで $30-50/日 |

### トークン構成の実態

あるパワーユーザーの分析（8ヶ月、10B トークン）:

| カテゴリ | 比率 |
|---------|------|
| キャッシュ読取 | **90%以上** |
| キャッシュ書込 | 約 6% |
| 入力/出力トークン | 合計 1% 未満 |

この構成が Max プランの圧倒的なコスト優位性の理由。キャッシュ読取が生の入力トークンとして課金されていれば $67,500 → 実際は $6,750。

### チームでの成功事例

6人の開発チームが8週間で API コストを **$2,400 → $680 に削減（72% 削減）**:
1. Sonnet をデフォルトモデルに変更
2. タスク間でコンテキストをクリア
3. `.claudeignore` の活用
4. 計画モードの徹底

### コミュニティ推奨ベストプラクティス

1. `/compact` を「定期メンテナンス」として使う（最後の手段ではなく）
2. `.claudeignore` は最初に設定する（トレードオフなしの最速改善）
3. 「説明不要、コードだけ」で出力トークン 60-70% 削減
4. Agent Teams は 1 エージェントの約 7 倍のトークンを消費 — チームサイズを小さく保つ
5. `DISABLE_NON_ESSENTIAL_MODEL_CALLS=1` で背景モデル呼び出しを無効化
6. サブスクリプションユーザーはモデル選択がコストに影響しないため、遠慮なく Opus を使ってよい
