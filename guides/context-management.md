# コンテキスト管理チートシート

> Phase 1〜3 の調査に基づく推奨。Phase 4B の実験で検証予定。

## コンテキストウィンドウの基本

Claude Code のコンテキストウィンドウは 200K トークン。使用率が上がると品質が劣化する。

```
[===== 使用中 =====>                     空き                     ]
0%              40%         60%              95%              100%
                 ↑           ↑                ↑
           注意開始    /compact 推奨    自動圧縮（遅い）
```

### 重要な閾値

| 使用率 | 状態 | 対応 |
|--------|------|------|
| 0〜40% | 正常 | そのまま作業 |
| 40〜60% | 注意 | 大きなファイル読み込みを避ける |
| **60%** | **圧縮推奨** | **`/compact` を実行**（何を保持するか指示） |
| 80%+ | 品質劣化 | 初期の指示を忘れ始める |
| 95% | 自動圧縮 | システムが自動で圧縮（33〜45K トークンのバッファ） |

## セッション開始時のコスト

何もしなくても以下のトークンが消費される:

| 項目 | トークン数 |
|------|-----------|
| システムプロンプト | 約 4,200 |
| CLAUDE.md（プロジェクト） | 約 1,800 |
| CLAUDE.md（グローバル） | 約 320 |
| 自動メモリ（先頭 200 行） | 約 680 |
| **合計** | **約 7,850** |

## 3 つのコンテキスト管理ツール

### `/clear` — 完全リセット

```
使う場面:
- タスクを切り替えるとき
- 同じ問題に 2 回失敗したとき
- セッションが 3 時間を超えたとき

効果: 会話履歴を全消去、CLAUDE.md とシステムプロンプトを再読み込み
注意: 進行中の作業内容は失われる（必要なら先にメモを残す）
```

### `/compact` — 要約圧縮

```
使う場面:
- コンテキスト使用率が 60% に達したとき
- まだ同じタスクを続けたいとき

使い方:
  /compact                        ← 自動要約
  /compact 認証機能の実装状況を保持  ← 保持する情報を指示

効果: 会話を要約して圧縮、指定した情報を優先保持
注意: 要約の過程で細かい情報は失われる
```

### サブエージェント — 隔離された探索

```
使う場面:
- 大量のファイルを読む必要があるとき
- コードベース全体を探索するとき
- メインのコンテキストを汚したくないとき

効果: 別のコンテキストウィンドウで作業、結果のみ返す
圧縮率: 約 98%（6,100 トークン読み込み → 420 トークンで返却）
```

## 使い分けフローチャート

```
タスクが終わった？
├─ Yes → /clear して次のタスクへ
└─ No
    ├─ コンテキスト 60% 超？
    │   ├─ Yes → /compact（保持情報を指示）
    │   └─ No → そのまま続行
    ├─ 大量のファイルを読む必要？
    │   └─ Yes → サブエージェントに委譲
    └─ 同じ修正に 2 回失敗？
        └─ Yes → /clear してプロンプトを改善して再開
```

## コンテキスト節約のテクニック

### 1. 具体的なファイル指定

```
Bad:  「認証のバグを直して」（Claude が全ファイルを探索 → 大量消費）
Good: 「src/api/auth.ts の validateToken のバグを直して」（直接読み込み → 最小消費）
```

### 2. @参照で必要なファイルだけ読む

```
@src/api/auth.ts と @src/middleware/session.ts を見て、
セッション有効期限の処理を確認して
```

### 3. CLAUDE.md にアーキテクチャを記述

ファイル探索コストを削減する:

```markdown
## アーキテクチャ
- API ハンドラー: src/api/handlers/
- ビジネスロジック: src/services/
- データアクセス: src/repositories/
```

### 4. サブエージェントで探索を隔離

大規模な調査はメインコンテキストの外で実行:

```
このリポジトリ全体で deprecated な API の使用箇所を調査して。
（→ Claude がサブエージェントを使って探索、結果のみ返却）
```

### 5. /btw で一時的な質問

本題と無関係な質問で会話を汚さない:

```
/btw TypeScript の Record 型の構文を教えて
```

## CLAUDE.md = 唯一の永続ストレージ

`/compact` や自動圧縮の後でも確実に残る情報は **CLAUDE.md に書いた内容だけ**。

セッション中に伝えた指示は圧縮で失われる可能性がある。常に適用したいルールは CLAUDE.md に記述する。
