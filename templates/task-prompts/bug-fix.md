# バグ修正プロンプトテンプレート

> Phase 2 の調査（09-task-bug-fix）に基づく推奨パターン。Phase 4B の実験で検証予定。

## 基本フォーマット

```
{{症状の具体的な説明}}

**再現手順:**
1. {{手順 1}}
2. {{手順 2}}
3. {{手順 3}}

**期待する動作:** {{正しい動作}}
**実際の動作:** {{現在の動作}}

**エラーログ:**
```
{{スタックトレースやエラーメッセージの全文}}
```

**関連ファイル:** {{ファイルパスがわかれば指定}}
```

## 使い分け: 3 つのパターン

### パターン 1: 原因が明確な場合（直接指示）

```
`src/api/auth.ts` の `validateToken` 関数で、期限切れトークンに対して
401 ではなく 500 が返る。

エラーログ:
```
TypeError: Cannot read property 'exp' of undefined
  at validateToken (src/api/auth.ts:42)
```

期待動作: 期限切れトークンには 401 Unauthorized を返す。
修正後、既存テストが通ることを確認してください。
```

### パターン 2: 原因が不明な場合（分析→修正）

```
ユーザーから「ログイン後にダッシュボードが表示されない」と報告がある。

**再現手順:**
1. /login にアクセス
2. 正しい認証情報でログイン
3. リダイレクト後に白い画面が表示される

**エラーログ（ブラウザコンソール）:**
```
Uncaught TypeError: Cannot read properties of null (reading 'role')
```

まず原因を分析してください。修正案を提示してから実装に進んでください。
```

### パターン 3: TDD バグ修正（テスト先行）

```
`calculateDiscount` 関数で、100% 割引クーポンを適用すると
合計金額がマイナスになるバグがある。

以下の手順で修正してください:
1. まずバグを再現する失敗テストを書く
2. テストが失敗することを確認
3. 最小限のコード変更で修正
4. テストが通ることを確認
5. 既存テスト全体を実行して回帰がないことを確認
```

## プロンプトのチェックリスト

- [ ] 症状が具体的（「動かない」ではなく何がどう動かないか）
- [ ] エラーメッセージ・スタックトレースを省略せず記載
- [ ] 再現手順がある（または再現条件を説明）
- [ ] 期待する動作を明記
- [ ] 検証方法を指定（テスト実行、特定の入力での出力確認など）

## 避けるべきパターン

| やりがち | 改善 |
|----------|------|
| 「ログインのバグを直して」 | 症状・エラーログ・再現手順を含める |
| エラーログの一部だけ貼る | スタックトレース全文を貼る |
| 修正後の確認方法を指定しない | 「テストを実行して確認」を付ける |
| 同じバグを 2 回以上修正指示 | `/clear` してより具体的なプロンプトで再開 |

## 根拠となる調査

- 3 要素の公式（エラー全文 + 場所 + トリガー）: Phase 2: 09-task-bug-fix
- TDD バグ修正パターン: Phase 2: 09-task-bug-fix, Phase 3: 13-best-practices
- 2 回失敗したら `/clear`: Phase 3: 14-anti-patterns
