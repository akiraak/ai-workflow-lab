# CLAUDE.md

このファイルは Claude Code がリポジトリを扱う際のガイドラインです。

## プロジェクト概要

AIを普段の開発・作業ワークフローに組み込む手法を調査・検証するリポジトリ。

## 言語・コミュニケーション

- ユーザーとのやり取りは日本語で行う
- コミットメッセージは日本語で書く
- コード中のコメントは英語でも日本語でも可（文脈に合わせる）

## コーディング規約

- 検証コードであっても、再現性を意識して書く
- 依存関係がある場合は requirements.txt や package.json で管理する
- 実験ごとにディレクトリを分けて整理する

## タスク管理

- **TODO.md** — 作業予定・進行中のタスクを `- [ ] タスク内容` 形式で管理する
- **DONE.md** — 完了したタスクを `- [x] タスク内容 (完了日: YYYY-MM-DD)` 形式で記録する
- タスク完了時は TODO.md から DONE.md へ移動する

## Web キャッシュ

Web 調査時はページ全文を `research/cache/` 以下にキャッシュする。

- セッション（実行タイミング）ごとに `research/cache/YYYY-MM-DDTHHMMSS/` ディレクトリを作成する
- 各ページは `{連番}_{ドメイン名簡略}.md` として保存する（例: `001_docs-anthropic-com.md`）
- 各ファイルの先頭に frontmatter で `url` と `fetched_at` を記録する
- セッションディレクトリ内に `_index.md` を作成し、URL とファイル名の対応表を管理する
- 同じ URL でも実行日時が異なれば別セッションに保存する（内容の変化を追跡可能にする）

## Git 運用

- ブランチ: main ブランチで直接作業（個人実験リポジトリのため）
- コミット: 変更の意図がわかるメッセージを付ける
